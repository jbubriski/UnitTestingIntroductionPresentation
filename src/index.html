Unit Testing

John Bubriski

What is unit testing?

Running isolated pieces of code and verifying an expected outcome.

Why do we unit test?

Make sure your code actually works!

Unit testing is your insurance policy.

Let QA focus on important things!

What do we unit test?

As much as we easily can.

How do we unit test?

Eliminate dependencies

  • Network connections
  • Databases
  • Web Services

Write the test

  • Name it well
  • Don't copy paste
  • Only do what you need

Make an assertion

  • One assertion per test
  • Wathc it fail
  • Refactor

Run the test

  • NCrunch
  • xUnit test runner
  • Resharper (w/plugin)

Examples!

  • Write many unit tests.
  • Make assertions.
  • Run your tests often.
